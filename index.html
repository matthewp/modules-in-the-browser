Solves

  • Don't have to care about order, just declare dependencies in each module.
  • Circular dependencies are possible.

Problems

  • Usually requires configuration of where modules are located.
  • People hate wrapping their code.

ECMAScript 6 modules: the final syntax

export


const math = { add, subtract };

export const a = "b";

export function add(a){
  a + 1;
}

function subtract(a){
  return a - 1;
}

export default math;

export { subtract, add as addOne };

import


import { add, subtract } from "math";

import { add as addOne } from "math";

import * as math from "math";

import math from "math";
import { default as math } from "math";

import math, { add } from "math";

import "math";

What is a Loader?

  1. System for loading and executing modules.
  2. Dependency manager.
  3. Module registry.

  • Browserify: NO
  • WebPack: YES (sort of)

Reflect.Loader


Reflect.Loader; // Capital L

Reflect.loader

  • Default Browser Loader.
  • Loads <module>s
  • Can be ignored.
  • Can be extended.

Provides hooks to extend the loading process.
